# Pass 179 — ReDMCSB inventory slot-box consolidation

Source-first audit:

- `DEFS.H:778-817` defines slots `0..37`, including inventory `0..29` and chest `C30_SLOT_CHEST_1` through `C37_SLOT_CHEST_8`.
- `DEFS.H:1873-1878` defines slot-box constants:
  - `C08_SLOT_BOX_INVENTORY_FIRST_SLOT = 8`
  - `C38_SLOT_BOX_CHEST_FIRST_SLOT = 38`
  - `M070_HAND_SLOT_INDEX(slotboxindex) ((slotboxindex) & 0x0001)`
- `DATA.C:977-1023` maps PC/F20+ `G0030_as_Graphic562_SlotBoxes[46]` to zone IDs:
  - status slot boxes `0..7` -> `C211..C218`
  - inventory slot boxes `8..37` -> `C507..C536`
  - chest slot boxes `38..45` -> `C537..C544`
- `DATA.C:1049-1087` maps slot allowed masks for the 30 inventory slots and 8 chest slots.
- `COMMAND.C:419-450` maps inventory mouse commands `C028..C057` to zones `C507..C536`.
- `COMMAND.C:499-506` maps chest mouse commands `C058..C065` to zones `C537..C544`.
- `COMMAND.C:2174-2177` dispatches commands `C028..C065` as `command - C020_COMMAND_CLICK_ON_SLOT_BOX_00...`, i.e. `command - 20` -> slot-box index.
- `CHAMPION.C:677-692` maps slot-box index to either status champion hands or inventory/chest storage:
  - `< 8`: champion index `slotBox >> 1`, hand slot `M070_HAND_SLOT_INDEX(slotBox)`
  - `>= 8`: inventory champion, slot index `slotBox - 8`; `>= C30_SLOT_CHEST_1` uses `G0425_aT_ChestSlots`.

Firestaff implementation:

- `inventory_slotbox_pc34_compat.[ch]` exposes the source-locked slot-box -> slot/command/zone/mask mapping and command dispatch route.
- This consolidates inventory and chest slot-box facts into one probeable module.
