# DM1 all graphics phase 2997-3016 — V1 side contents normal path

## Change

Normal V1 now draws side-lane open-cell contents (floor items, creatures, and projectiles) in the source viewport path instead of leaving that code reachable only through the debug/prototype side-feature renderer.

The new pass:

- draws only contents, with no invented side-pane fill or border;
- reuses the existing source-backed C2500 object placement, C3200 side-creature placement, object pile shifts, creature transparency, and projectile placement helpers;
- runs after source wall/door/ornament/button passes, before center-lane object/creature/projectile contents;
- stops at nearer non-open center-line blockers so side contents do not paint through a closed center wall/door.

## Source evidence

- `DUNVIEW.C` / `F0115` draws visible square contents through the same viewport object/creature placement pass for front and side view cells.
- Layout-696 source helpers already present in Firestaff are used:
  - `m11_c2500_object_zone_point()` for C2500 object positions;
  - `m11_c3200_creature_side_zone_point()` for C3200 side creature positions;
  - `m11_draw_projectile_sprite()` for M613/G0210 projectile aspects.

## Visual effect

Side-cell loot, side creatures, and side projectiles can now appear in the normal DM1/V1 viewport without enabling debug HUD. This narrows the visible gap where the source wall/door viewport was present but live side-lane occupants vanished.
